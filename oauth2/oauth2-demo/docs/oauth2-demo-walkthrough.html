The whole demo lives in one Spring Boot process on :8085 — it plays Authorization Server, Resource Server and OAuth2 Client at once — talking to a React SPA on :3000. Step through the real request/response sequence below, exactly as it happens on the wire.

This walks through exactly what the sample app does: PKCE generation, login + consent, the token exchange, persistence to H2, and the final call to a protected resource.

Run the real thing (not just this walkthrough)
Terminal 1 — backend
.\gradlew.bat :oauth2-demo:bootRun --no-daemon
# → http://localhost:8085
Terminal 2 — frontend
cd oauth2/oauth2-demo/frontend
npm install
npm start
# → http://localhost:3000
